# Security Policy

## Supported Versions

Security fixes are prioritized for the latest published release.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Report vulnerabilities privately via GitHub Security Advisories:

- Go to the repository `Security` tab.
- Click `Report a vulnerability`.
- Provide impact, reproduction steps, and affected versions.

If Security Advisories are unavailable, open a private communication channel with the maintainers and include the same details.

## Response Process

- We will acknowledge receipt as soon as possible.
- We will validate and triage severity.
- We will work on a fix and coordinate a responsible disclosure timeline.

## Scope

This policy applies to source code and release artifacts in this repository.
